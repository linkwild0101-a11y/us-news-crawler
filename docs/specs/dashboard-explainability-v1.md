# Dashboard Explainability v1（规格草案）

- 日期: 2026-02-23
- 版本: v1
- 目标: 把“信号-机会-证据”解释链条标准化，降低决策理解成本

## 1. 解释层核心对象

1. 指标解释（Tooltip / 字典中心）
2. Why-Now 说明
3. 证据列表（热点、实体关系、来源链接）
4. 数据质量提示（翻译状态、新鲜度、来源健康）

## 2. v1 展示规则

- 用户可见文本优先中文
- 英文证据无中文时显示“翻译处理中”
- event_type 使用中文标签映射展示

## 3. 质量徽章（建议）

- `fresh`: 新鲜数据
- `degraded`: 部分源降级
- `critical`: 关键源异常

## 4. 回退策略

- 翻译失败不阻断看板
- 证据缺失时展示安全占位文案
